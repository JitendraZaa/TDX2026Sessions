### About This Session

Learn to design multi-org architectures using the home-companion model in Data Cloud One and patterns for data sharing, governance, and unified profiles across 10 or more Salesforce orgs.

**Learning Objectives:**
1. Design a Data Cloud One architecture selecting home org and companion orgs based on data gravity and governance requirements
2. Configure data space sharing policies that enable unified profiles while respecting regional data sovereignty boundaries
3. Implement SDLC patterns including sandbox strategy and metadata synchronization for multi-org Data Cloud environments

### Data Spaces
**[Salesforce Data Spaces Documentation](https://help.salesforce.com/s/articleView?id=data.c360_a_data_spaces.htm&type=5)**

Learn how to create virtual environments that logically organize data based on specific themes, brands, regions, or departments. Administrators can control access and visibility of Data Lake objects, data models, and other Data Cloud metadata.
